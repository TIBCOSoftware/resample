Data and Statistic Details

This page gives details for the Data and Statistic fields as well as how to pass additional arguments to the statistic The fields are found on the "Model" tab for the Bootstrap, Jackknife and Influence, and Permutation Test menus.

Data Set

Specify the data to resample. This may be a vector, matrix, or data frame.

It may also be a model object, e.g. created by a call to lm. The object must contain a call component that indicates the name of the data originally used to create the object.

Statistic to Estimate

In the Expression field, specify the expression describing the statistic to be resampled. this may be a function that accepts data as the first argument, or an expression that calculates the statistic.

The statistic should normally return a numerical object, a vector or matrix. However, it may return NULL, when resampling for graphics rather than for numerical calculations.

Function

One choice is to give the name of a function, such as mean or colMeans.

If the Data Set is a model object, then the statistic should be a function like coef that operates on the model object.

If you need to pass arguments to the statistic, pass them using args.statistics in the Additional Arguments field. For example,
args.statistics=list(trim=.1)
If you want to resample a function for which the data is the second argument, then pass the first argument using args.statistics; the data is passed as the first first argument that is not passed by name using args.statistics.

Expression

Often it is more convenient to give an expression that calculates the statistic. For example,
mean(x, trim = .1)
calculates the trimmed mean of x.

Using an expression is particularly convenient when the data is a data frame, because you can refer to variables by name, without explicitly subscripting them. For example, if the data is fuel.frame, then you may use e.g.
mean(Fuel, trim = .1)
which is actually a shortcut for
mean(fuel.frame$Fuel, trim = .1)

If the Data Set is given by name, e.g. lottery.payoff, then just use its name, e.g.
mean(lottery.payoff, trim = .1)
However, if the Data Set is given as an expression that computes a data set, e.g. ethanol[1:45, "NOx"], then use the name data to refer to the data, e.g.
mean(data, trim = .1)

See also
